decide - Support Decision-Making through Collaboration

Björn Ebbinghaus

Introduction

We present a proposal on how to improve decision-making in groups by giving participants the tools to improve a set of candidates together.

As we have noticed in an experiment in 2019, involved participants in a decision not only want to express their own proposals, but also actively participate in the improvement of existing proposals. To reflect this, we use a technique common in software development, in which changes to a code base each have one (or more) predecessors on which they build.

We hope to move from commenting on why a proposal is not good or how it can be improved to creating new and improved proposals.
